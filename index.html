1. Arborenscense du projet

Ces dossiers, sous-dossiers et fichiers seront les bases de notre projet pour la pratique de Git.

3. Compte GitHub

Création sur GitHub

Suivre les étapes sur site pour l'enregistrement de votre compte.

La platform GitHub nous permitira héberger notre projet et disposer d'un écosystèmes des outils pour travailler le projet de manière collaboratif,  automatisé et surtout centralisé.


4. Git configuration

À réaliser avant la première utilisation
$ git config --global user.name "nom_utilisateur"
$ git config --global user.email "mail_utilisateur"

Changer nom_utilisateur pour votre nom d'utilisateur et mail_utilisateur avec le mail liée à la compte GitHub

Git est un outil collaboratif qui a besoin de déterminer qui sera l'utilisateur dans l'ordinateur, afin d'indentifier les modidication réalisées. Plusieurs peuvent êtres liées avec l'option --local pour réaliser un configuration par utilisateur.


5. Premier projet

Commandes de bases Git
$ git init

L'option init initialisera le projet à vertionner dans le dossier.

$ git add .

Une fois initialisé, l'option add . (ou */-A/"nom-de-fichier") ajoutera les nouveaux fichier ou ses modifications au stage du projet.

$ git status

L'option status imprimera l'état du projet. Les modification ou nouveaux fichiers seront visibles à cette étape.

$ git commit -m "message description du commit"

La savegarde se réalise avec l'option commit et l'option -m "message descriptif" fournie la description respectif aux modifications. Tout sera ajouté à l'historique.

$ git log

L'historique des versions s'imprime avec l'option log. Une liste avec descriptions, identifiants uniques et d'autres détails sera fournie.

D'autres options peuvent être mis en place mais les principales éléments ont été décrits dans ce résumé pour avoir un idée générale de comment travailler avec Git et supplémentairement GitHub pour son hébergement.
